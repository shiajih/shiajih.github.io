面試分享(Realtek瑞昱/Cadence益華電腦)

發表於 2019-03-17 | 分類於 雜記 |

背景介紹

119文組、114CS碩在學，需服四個月兵役
研究領域為Back-end的電子設計自動化(Electronic Design Automation - EDA)
主要用C/C++寫相關演算法

面試前置準備

碩士這一兩年，老師滿鼓勵我們寒暑假到CIC上相關EDA Tool課程。要走這領域，ICCompiler or ICC2、Innovus、Design Compiler都得是必會的工具，儘管研究不是100%會用到，不過去上個課拿證書也是挺不錯。

修課方面大概就是實體設計自動化(Physical Design Automation)和VLSI-Testing等，清大研究所的必修學分說多不多說少不少，但每堂課收穫都很豐富。

Coding部份大多是看書或者線上課程如Coursera、Udemy等。個人認為Coursera的質量頗高，我是在〈北京大學：程序設計與算法〉奠立C/C++基礎的。從計算機概論到演算法及資料結構，都講得鉅細靡遺。

書的話網路上大多推薦C的書是《The C Programming Language》
但這本我覺得，對完全沒有任何程式基礎的人讀起來可能有點霧煞煞，不知道為什麼可以這樣可以那樣。
C++的話則是英文版《C++ Primer 4/e》，不過目前應該可以看《C++ Primer 5/e》了，C++11提供不少好用語法及功能值得學習（但我是看簡中版，想說看中文比較快），還有《培養與鍛鍊程式設計的邏輯腦：世界級程式設計大賽的知識、心得與解題分享 [第二版]》

這本能看完並理解其中80%內容應該很強了。當然還有許多經典Coding以及C/C++的書籍，日後再專寫一篇文推薦。

還有一些有趣主題如Design Patterns、Advanced C++ programming、Verilog等在Udemy上的資源很豐富，一堂課特價時也不過300~400元，很划算，不過上課前最好聽一下開放預覽的片段，有些講師是印度人西班牙人，得看自己能不能習慣這些口音。

面試前兩三個禮拜有準備自介投影片，和複習C++常見面試題，以及再熟悉一下目前Project的內容及進度，畢竟還沒有畢業，也沒啥東西可以講。

Realtek 瑞昱 實體設計（總部）

這間是在104上主動投預聘，投完後隔天人資就打來邀約面試。
自傳、成績單什麼的準備齊全後就到現場，HR會問返程時需不需要搭車，在科技版上看到會補貼面試車馬費的公司好像不多，以這點來說瑞昱算不錯。

進到小房間後開始和一位年輕面試官面試，前半部份主要是他在講公司、這個職位在做什麼，後半部份開始自我介紹。那因為我背景比較特殊，所以面試官肯定會問為何想從文跳資工，這問題的答案已經想過很多次因此他問得時候很快能反應。
不過瑞昱是我第一家面試的公司，所以在講得過程中頗緊張，卡卡的。中間有講到目前做得Project(Routing相關)，有個地方回答不出來，面試官試圖引導我，但還是答不太出來…之後他就說好沒關係就讓我接著講下去。最後問了面試官一些職位上的問題後就結束。

我在修PDA的時候，關於Timing並沒有講太多，Floorplan、Placement、Routing等相關演算法佔絕大部分。個人覺得如果要應徵這個職位，一些Timing的東西要搞得很熟，如STA、Setup time、Hold time、Slack、Clock Tree、Skew、Power EM等。
兩個禮拜後致電問結果得到感謝函，其實沒有很意外，因為我真的準備不充足，又是第一間面試的公司因此表現得很緊張。

總結這次面試，沒有白板題沒有考卷，純粹在附的履歷以及投影片中問答。

Realtek 瑞昱 多媒體（二廠）

過了差不多一個月，再次收到瑞昱的面試通知，這次是在二廠，信中是寫實體設計工程師，但到了現場主管才說是多媒部門寫code的職位。這次面試不同的是一開始有做人格測驗，滿多題的但時限內一定寫得完。那因為有了上次的面試經驗，所以這次在心理上放鬆不少，比較能從容面對。主管一樣有問為何從文轉資工，然後大多介紹這職位的工作內容。大概是因為有第一次面試的經驗，這次面試不到一小時就結束了。儘管我在履歷上都有寫就學中且要未役，但直到面試最後，主管才知道我還沒畢業，且還要服四個月兵役，就發一張名片，希望服完兵役後再與他聯繫。

結果：無聲卡

Cadence 益華電腦

這間是學長內推的，首先會寫一張與程式及Boolean Logic有關的考卷，之後兩位主管面試，第一位問得問題個人覺得都很專業，如要怎麼確定A這件事的正確性？要怎麼衡量一個tool跑得結果是好還是不好？還有VLSI-testing上的，Stuck-at fault 1/0 要怎麼判斷？可以怎麼Formulate這問題？FSM的數學模型（因為我用FSM回答Stuck-at fault）？

EDA跟硬體脫離不了關係，所以主管有問會Verilog嗎？我說在網路上有自學過一點，他就接著問Blocking / Non-Blocking的差別，還有大概解釋一下testbench、4-bit Adder大概怎麼寫，主管有訝異且好奇，對於一個文組學生為何會想學這些，我回答說主要有兩個原因：

  1. 因為常聽到資工本科的同學在講Verilog，所以覺得這東西好像很重要，所以想去學。
  2. 在學校內的佈告欄上，Cadence貼了一張某個年份的徵才海報，第一個職位就是和硬體相關，我問旁邊同學這職位在幹嘛，大概要怎樣的技能，同學說首先你要會Verilog。

另外，我在Resume上有提到有在接網頁類的case，所以懂一些HTML/CSS、JS、MYSQL等，還有修課有用到一點點Python，所以有問這些語言的特性，和C/C++有啥差別。

第一位主管感覺起來人真的很好且很強，很有耐心聽你講，且會給你很多回饋，大概面了一個多小時，主管就說ok，他覺得面試滿愉快的。

第二位是部門主管，沒什麼特別問題，主要是閒聊以及介紹部門和底下各個Team的工作內容，當然也有問為何想從文跳資工。

包含寫考卷，大概面了四小時左右吧，這次面試真的是很愉快的經驗。

結果：口頭Offer，等核薪

後記

後來還有幾間公司(Synopsys、趨勢、華邦電、中磊等)的邀約面試，但因為Cadence有給口頭Offer（應該不會被婊吧QQ），就都婉拒了。
個人覺得面試重要的是保持冷靜，回答問題時可以想一下不要太快回答，縱使是不會的問題，也盡量想一套說法，還有投影片上的東西都要100%了解，不能一知半解，不了解的東西就不要放進去。

有空的時候可以多寫寫Leetcode，就算是Easy也沒關係，C的主管有問有沒有在寫Leetcode，我回答有在寫UVA，他建議我改寫Leetcode我也不知為啥。

這次面試心得就分享到這，希望有幫助到正在找工作的朋友們。
